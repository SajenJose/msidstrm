This utility lists / modifies the various data streams present in a Windows Installer Database (MSI Files). We can read the various data streams from the _Streams table inside the Windows Installer Database. Apart from the MSI Capabilities, this utility also comes with some small goodies like GUID Conversion into squished format and vice versa, cab file operations and also find the file / directory path size or length.

Usage:

-: Read Data Streams: msidstrm.exe <msifilename>.msi

-: Add Data Streams: msidstrm.exe /a <msifilename>.msi <newfile>.ext

-: Delete Data Streams: msidstrm.exe /d <msifilename>.msi <datastream name>

-: Extract Data Streams: msidstrm.exe /eas <msifilename>.msi

-: Extract Files: msidstrm.exe /eaf <msifilename>.msi

-: Convert GUID: msidstrm.exe /guid [N/P/B/D] {guid}

-: Process Cab Files: msidstrm.exe [/lcab | /ccab | /ecab] <cabfilename> <directory>

-: Get File Path Length: msidstrm.exe /gfp "<path / directory - to search>"

-: Get CPU Arch: msidstrm.exe /cpu

 

Options:

/a - Add a data stream to the msi file

/anb - Add a data stream to the msi file, without backing up the original msi file

/d - Delete a data stream from the msi file

/dnb - Delete a data stream from the msi file, without backing up the original msi file

/eas - Extract all streams and table information from the msi file

/eaf - Extract all files from the mi file

/guid - Converts GUID to compressed / packed / squished and vice versa

/lcab - Lists cab file contents

/ccab - Creates a cab file from the contents of the specified directory

/ecab - Extracts the files from the cab file

/gfp - Lists or Saves the Files and Folder Path Size / Length into a comma separated file.

/cpu - Gets the CPU Architecture like 32-bit or 64-bit

/? - Displays the help / usage message
